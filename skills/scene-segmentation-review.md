# Skill: 场景切分审核 (Scene Segmentation Review)

## 审核规则
1. **无重叠**：相邻场景的 source_text 不得有重复内容。计算重叠词数占比，若超过 30% 则视为重叠，标记为错误。
2. **线性顺序**：场景 ID 应按叙事顺序递增，且场景内的事件必须按原文时间线，不可倒置。
3. **完整性**：所有场景的 source_text 拼接后应覆盖原文的全部叙事内容，无重大遗漏。
4. **临时角色注册**：如果场景 summary 中包含“临时角色：TEMPxx-角色名”，应检查这些临时角色是否已在场景中实际出现，若未出现则提示。
5. 地点一致性检查
- **单一地点规则**：检查每个场景的 `source_text` 中是否出现了除 `location_ref` 指定地点之外的其他地点（通过已知地点名称列表匹配或上下文判断）。
  若出现其他明确地点，则标记为“场景包含多个地点，违反单一空间锚点原则”。
- **交叉剪辑检测**：若某个场景的 `source_text` 在短时间内（如相邻段落）反复切换空间，标记为“疑似交叉剪辑，应拆分为多个单地点场景”。
- 修正指引：将该场景按地点拆分，分别创建新场景，确保每个场景的 `source_text` 只包含各自地点的内容。

## 修正指引
- 重叠时，明确保留前一个场景的 source_text，将后一个场景中重叠部分删除或重新分配边界。
- 遗漏时，要求添加新场景或扩展已有场景的 source_text。
- 若发现多地点：将该场景按地点拆分，分别创建新场景，并确保新场景的 `source_text` 只包含各自地点的内容。
- 若为交叉剪辑：将交替的段落按地点分组，生成两个或多个连续的单地点场景。不要创建跨地点的混合场景。
