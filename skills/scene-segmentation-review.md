# Skill: 场景切分审核 (Scene Segmentation Review)

## 审核规则
请逐项检查以下规则，每发现一处违规记录一条 issue，并给出具体修正指引。

### 1. 空间锚点唯一性
- **检查**：每个场景的 `source_text` 是否只发生在 `location_ref` 指定的空间内。
- **违规标记**：若出现其他明确地点名称，标记为 `场景包含多个地点，违反空间锚点原则`。
- **修正指引**：将该场景按地点拆分为多个独立场景，确保每个新场景的 `source_text` 只包含各自地点的原文内容。

### 2. 交叉剪辑检测
- **检查**：若某个场景的 `source_text` 在相邻段落间反复切换空间（如 A→B→A→B），标记为 `疑似交叉剪辑`。
- **修正指引**：将交替的段落按地点分组，生成两个或多个连续的单地点场景，按出现顺序排列，禁止合并为跨地点场景。

### 3. 事件区间不重叠
- **检查**：所有场景的 `summary` 是否有重复或高度相似的事件描述。
- **违规标记**：若两个或以上场景 summary 描述同一事件画面，标记为 `事件唯一性违规，场景复述`。
- **修正指引**：合并重复场景，只保留一个完整呈现；其他场景仅保留各自独有空间内的内容，若无独有内容则直接删除。

### 4. 严禁概览式冗余
- **检查**：是否存在某个场景的 `source_text` 概括了全文或后续场景的全部事件（“先概述后展开”的冗余场景）。
- **违规标记**：若某场景的 beats 覆盖了后续多个场景的事件区间，标记为 `概览式冗余场景，违反线性推进原则`。
- **修正指引**：直接删除该冗余场景，将其他场景按原文顺序重新编号。

### 5. 线性顺序与完整性
- **检查**：场景 ID 是否按叙事顺序递增；相邻场景的 `source_text` 是否无缝衔接、无重复或遗漏。
- **违规标记**：若顺序错乱或存在内容重叠/遗漏，标记为 `线性顺序错误` 或 `内容重叠/遗漏`。
- **修正指引**：调整场景顺序，裁剪重叠部分，补充遗漏内容。

### 6. 临时角色注册
- **检查**：若 scene `summary` 中包含“临时角色：TEMPxx-角色名”，该临时角色是否在 `source_text` 中实际出现。
- **违规标记**：若未出现则标记为 `临时角色未实际出现`。
- **修正指引**：移除未出现的临时角色标注，或补充对应原文片段。
